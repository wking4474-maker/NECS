## 配置要求
- 宿主机 Linux 镜像: `Rocky-10.0-aarch64-dvd1.iso`
- 宿主机具体版本：`Rocky Linux release 10.0 (Red Quartz)`
- 硬件要求：8个vcpu，16G内存，硬盘大小：60G
- 安装pip：`yum install pip`

## AI部署
### 安装ollama
#### 将 ollama-linux-arm64.tar.zst 解压缩，使用指令如下：
- 首先安装 zstb: `yum install zstd`
- 解压缩文件: `tar --use-compress-program=unzstd -xvf ollama-linux-arm64.tar.zst`
- 这里我们将ollama解压缩到 /opt 目录下
#### 将 部署qwen2.5 大模型到ollama
- 解压缩大模型：`tar xvf qwen2.5-1.5b.tar`
- 将大模型移动到ollama存放模型的路径：`mv models/ /root/.ollama/`
- 启动ollama: 
```shell
cd /opt/bin
OLLAMA_HOST=0.0.0.0:11434 ./ollama serve & #这里是放在后台启动
#用这种方式启动是因为没有做systemctl
```
- 测试大模型:
```shell
cd /opt/bin
./ollama run qwen2.5:1.5b

#如果部署成功，终端如下：
[root@localhost bin]# ./ollama run qwen2.5:1.5b
>>> Send a message (/? for help)
#这里可以输入一些问题来测试大模型是否成功部署
```

### 部署dify
#### 安装 docker 
- `docker`的`rpm`安装包在 `docker-ce10-arm64`文件夹中，将这个文件夹拷贝到宿主机，利用yum安装
### dify配置
- 将`dify-1.11.4.zip`复制到宿主机的`/opt`目录下,并利用 `unzip` 解压缩
- 将`dify-images` 复制到宿主机的`/opt`目录下，该文件存dify需要用到的所有容器
- 将`dify-images`下的所有容器镜像导入到docker中：
```shell
   cd /opt/dify-images/
   docker load -i busybox_latest.tar
   docker load -i dify-api_1.11.4.tar
   docker load -i dify-plugin-daemon_0.5.2-local.tar
   docker load -i dify-sandbox_0.2.12.tar
   docker load -i dify-web_1.11.4.tar
   docker load -i nginx_latest.tar
   docker load -i postgres_15-alpine.tar
   docker load -i redis_6-alpine.tar
   docker load -i squid_latest.tar
   docker load -i weaviate_1.27.0.tar
```
- 利用`docker compose`启动dify
```shell

   cd /opt/dify-1.11.4/docker
   cp .env.example .env #复制环境配置文件，这步很重要
   
#修改环境配置文件，在文件末尾添加一下三行:
FORCE_VERIFYING_SIGNATURE=false		
# Dify 平台允许安装所有未在 Dify Marketplace 上架（审核）的插件
PLUGIN_MAX_PACKAGE_SIZE=524288000 	
# Dify 平台允许安装 500M 大小以内的插件
NGINX_CLIENT_MAX_BODY_SIZE=500M 	
# Nginx客户端将允许上传 500M 大小以内的内容

   
   #启动dify
   docker compose up -d
```
